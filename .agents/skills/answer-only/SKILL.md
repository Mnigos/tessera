# Answer Only

Use this skill when the user wants a direct answer and has explicitly ruled out implementation or file edits.

## Rules

- Answer directly and concisely.
- Do not edit files.
- Do not create commits, branches, GitHub comments, or discussions.
- Do not run tools unless needed for accurate current or repo-specific facts.
- If inspection is needed, read only the minimum context required and report the answer.
- If the question exposes a bug or improvement opportunity, mention it only as an answer; do not fix it.
